# 功能设计
## 文件
- prepare_data.py

## 要求
- 修改代码的同时，同步更新 README.md 和对应的设计文档

## 功能描述
- 下载后续导入流程所需的数据文件
- 支持指定下载 URL
- 支持指定下载目录和输出文件名
- 支持覆盖控制
- 支持超时配置和下载进度输出
- 支持将下载后的 HDFS JSON / JSON.GZ 日志文件转换为 CSV

## 实现细节
- 脚本文件为 `prepare_data.py`
- 当前阶段以“下载文件”为主要功能，先搭好 CLI 和下载执行框架
- 命令行默认无参数时输出 help 信息
- 默认下载地址为：
```
https://quickwit-datasets-public.s3.amazonaws.com/hdfs-logs-multitenants.json.gz
```
- 默认下载目录为 `data`
- 可通过位置参数 `url` 覆盖默认下载地址
- 支持的可选参数如下：
  - `--download`，指定后执行下载动作
  - `--output-dir`，默认 `data`
  - `--output-file`，默认空，未指定时从 URL 路径自动推断文件名
  - `--overwrite`，指定后允许覆盖已存在文件
  - `--timeout`，默认 `30` 秒
  - `--progress-interval`，默认 `3` 秒，表示每隔多少秒输出一次下载进度
  - `--chunk-size`，默认 `65536` 字节，即 64 KiB
  - `--input-file`，默认空，未指定时转换输入使用下载得到的文件
  - `--convert-out`，默认空，指定后将输入文件转换为 CSV
  - `--max-rows`，默认 `0`，表示转换时不限制行数
  - `--convert-batch-size`，默认 `50000`
  - `--dry-run`，只输出下载计划，不真正执行下载
- 脚本通过动作参数控制行为：
  - `--download` 表示执行下载
  - `--convert-out` 表示执行转换
  - 两者可以同时指定，表示先下载再转换
  - 若两者都未指定，则直接报错退出
- 下载执行流程如下：
  - 解析 URL 和输出路径
  - 创建目标目录
  - 若输出文件已存在且未指定 `--overwrite`，则直接报错退出
  - 下载数据到临时文件 `<output>.part`
  - 下载成功后将临时文件原子替换为目标文件
  - 下载失败时清理临时文件
- 下载时使用固定大小的分块读取
- 进度输出按时间间隔控制：
  - 当达到 `--progress-interval` 指定的秒数间隔时，输出当前已下载字节数
  - 若响应头中包含总大小，则同时输出总大小
  - 下载进度使用同一行覆盖更新的方式输出，不逐行追加打印
  - 下载刷新按计划时间点触发，并优先使用 `read1()` 和更细粒度分块读取来降低刷新延迟
  - 下载结束后输出最终下载字节数和总耗时
- 转换执行流程如下：
  - 确定转换输入文件，优先使用 `--input-file`，否则使用下载输出文件
  - 当指定 `--input-file` 且未指定 `--download` 时，跳过下载流程，直接进入转换
  - 当输入文件后缀为 `.gz` 时，使用 gzip 文本方式读取；否则按普通文本读取
  - 逐行解析 JSON 日志
  - 以批量方式写出为 CSV，字段固定为 `timestamp,severity_text,body,tenant_id`
  - 当达到 `--max-rows` 上限时停止转换
  - 转换输出先写入临时文件，再原子替换到目标 CSV 文件
  - 转换时按 `--progress-interval` 输出累计转换行数和耗时
- 参考 `~/pingcap/tici/ci/read_hdfs.py` 中的 HDFS 日志读取和 CSV 输出方式
- 当 URL 无法推断文件名且未指定 `--output-file` 时，脚本直接报错退出
- 当参数非法、网络请求失败、文件写入失败时，脚本直接报错退出
